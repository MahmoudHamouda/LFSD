# Governance

## Current model
LFSD is maintained by Mahmoud Hamouda (@MahmoudHamouda), who reviews and
approves all changes to the main branch. Development uses AI-assisted
engineering tools; all merges occur under maintainer review.

## Decision-making
Changes are proposed via GitHub issues and pull requests. Design decisions,
scope questions, and roadmap proposals are discussed in the open on the
issue tracker. Substantive decisions are documented in the relevant issue
or in `docs/`.

## Intended evolution
The project is designed for multi-stakeholder stewardship: the goal is
governance under a nonprofit or public-benefit structure in which no single
commercial entity controls the codebase or roadmap. As independent
contributors and adopting institutions join, maintainership will broaden
accordingly. This document will be updated as that structure forms.

## Contributions
Contributions are welcome via pull request. All contributions are reviewed
for consistency with the design principles in the README (policy-first
architecture, no fabricated data, governance as architecture) and must pass
automated security checks (see `SECURITY.md`).
